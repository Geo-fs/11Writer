## Reporting Shift

11Writer is now better framed as an evidence-aware reporting desk than as broad passive monitoring.

That means:

- broad monitoring still matters for intake and awareness
- the main product value is question-driven, source-backed reporting
- current work should favor report inputs, export-safe synthesis, validation closure, and real missing coverage
- current work should not reopen already implemented sources as if they were fresh intake
- the completed reporting-helper wave now exists to support a fresh bounded source-expansion wave, not to justify more helper renames

## Broad Monitoring Vs Reporting

| Mode | What it is for | What already exists | What to avoid |
| --- | --- | --- | --- |
| `broad monitoring coverage` | intake, source health, anomaly/review surfacing, family overviews | environmental overviews, marine context summaries, aerospace review/readiness helpers, Data AI family review/readiness/recent-items, webcam source-ops surfaces, Source Discovery candidate/review runtime | treating monitoring presence as proof of truth, urgency, impact, or action need |
| `question-driven reporting inputs` | answering a place/entity/timeframe/source-family question with source-backed evidence and caveats | export packages, review queues, report-safe lines, fusion-snapshot helpers, source-health summaries, evidence-basis tagging, source-mode tracking | reopening already built connectors instead of composing existing evidence |
